# 02 — План внедрения строительных калькуляторов

> Продакшн-площадка: `komplid-marketing` — раздел `/kalkulyator` уже существует
> (3 калькулятора: аванс, КС-2 НДС, рабочие дни) с готовой архитектурой.
> Этот план расширяет раздел до полноценной сетки инструментов, связанной с СП-страницами.
> Стратегия и blindspot-реестр: `00-STRATEGY-SEO-AEO-GEO.md` (особенно B-4 — данные только вручную!).

---

## 1. Зачем калькуляторы (сверх очевидного трафика)

1. **Линкбилдинг**: инструменты — единственный тип контента, на который сайты и форумы ссылаются добровольно.
2. **AI-рекомендации (GEO)**: ассистенты советуют инструменты («посчитать можно на …») — это
   трафик, не зависящий от позиции в выдаче.
3. **Поведенческие сигналы**: время на странице и возвраты у калькуляторов на порядок выше, чем у статей —
   важно для Яндекса.
4. **Семантический граф**: калькулятор ↔ СП-страница ↔ шаблон ↔ статья взаимно усиливают друг друга.
5. **Мягкие лиды**: «сохранить/распечатать расчёт» за email.

## 2. Архитектура (переиспользуем готовый паттерн komplid-marketing)

Уже существует и менять не нужно:
- `src/lib/calculators-data.ts` — массив `CALCULATORS` (slug, SEO-метаданные, FAQ, описание);
- `src/components/calculators/<Name>Calculator.tsx` + чистая логика в `logic.ts` (+ `logic.test.ts`);
- `src/app/kalkulyator/[slug]/page.tsx` — SSG из массива, JSON-LD, FAQ;
- регистрация в `sitemap.ts`.

Дополнения к паттерну для новой волны:

1. **Константы из нормативов** — отдельным блоком в `logic.ts` с комментарием-источником:
   ```ts
   // Источник: СП 20.13330.2016, таблица 10.1 (снеговые районы)
   export const SNOW_LOAD_BY_REGION = { I: 0.5, II: 1.0, III: 1.5, ... } as const;
   ```
   Никогда не парсим собственные СП-страницы (blindspot B-4). Каждая таблица констант —
   с юнит-тестом на контрольные значения из текста СП.
2. **Блок «Нормативное обоснование»** на странице калькулятора — ссылки на конкретные пункты
   СП-страниц (`/normativ/sp-20-13330-2016#p-10-1`) — уникальная фича против конкурентов-калькуляторов.
3. **Постоянная ссылка на расчёт** — состояние в URL-параметрах (`?s=III&roof=flat`) →
   шарится в чатах, индексируется как long-tail.
4. **HowTo schema** («Как рассчитать…») в дополнение к FAQPage.
5. **Embed-режим** (`/kalkulyator/<slug>/embed`, волна 3) — упрощённый вид для iframe с
   обязательной ссылкой-атрибуцией на полную версию.

### Идеи-парковка (волна 4+, по спросу)
Пересчёт марки/класса бетона (М↔В), осадка фундамента (упрощённо), расход электродов на сварку,
опалубка, стоимость демонтажа, калькулятор НМЦК/индексов сметной стоимости (осторожно: данные
меняются ежеквартально), проверка процента армирования, шаг стропил.

## 4. Структура страницы калькулятора (контент-стандарт)

1. H1 + прямой ответ («Калькулятор X рассчитывает Y по методике Z — бесплатно, без регистрации»).
2. Сам виджет (выше сгиба, работает без единого клика по вводу дефолтов).
3. «Как считается» — формула + описание переменных (текст, доступный краулерам!).
4. «Нормативное обоснование» — пункты/таблицы СП со ссылками на /normativ/….
5. Пример расчёта с числами (AEO любит конкретику).
6. FAQ ≥ 5 вопросов (FAQPage schema) + HowTo schema.
7. Связанные калькуляторы и шаблоны, контекстный CTA Komplid (для профильных — /smetchik | /pto | /prorab).

## 5. Каталог /kalkulyator (хаб)

Группировка по категориям: «Деньги и договоры» / «Материалы и объёмы» / «Инженерные расчёты по СП».
Каждая категория — свой H2 + описание (SEO-текст). ItemList schema. Счётчик «N бесплатных
калькуляторов для строительства» в title хаба.
