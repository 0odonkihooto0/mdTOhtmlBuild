# pdf-to-html — Pipeline для конвертации PDF в красивые HTML-страницы

Превращает PDF-документы (нормативы, регламенты, отчёты, статьи) в
самодостаточные HTML-страницы в стиле [effective-html](https://github.com/plannotator/effective-html)
с дизайн-токенами [Komplid](https://github.com/0odonkihooto0/Komplid2).

## Что делает pipeline

1. **Извлекает текст** из PDF через PyMuPDF (`get_text`)
2. **Извлекает таблицы** через `find_tables()` — с заголовками и строками
3. **Извлекает рисунки** — находит подписи «Рисунок N.M», определяет область
   графика (векторные пути + inline-растры) и рендерит crop страницы в PNG
4. **Объединяет строки в абзацы** (PyMuPDF выдаёт текст построчно)
5. **Определяет разделы** (1, 2, 3... + «Предисловие», «Введение», «Библиография»)
6. **Определяет термины** (для секций «Термины и определения»)
7. **Генерирует HTML** — один самодостаточный файл со встроенными картинками (base64)

## Возможности итогового HTML

- **Komplid design tokens**: oklch-палитра (slate + steel-amber), Inter + JetBrains Mono
- **Dark mode**: кнопка переключения, запоминание в localStorage, авто-определение `prefers-color-scheme`
- **Sticky TOC** слева со всеми разделами
- **TL;DR-блок** — первые 2-3 предложения документа
- **Иерархические `<details>`**: разделы → подразделы (5.1) → под-подразделы (5.2.1)
- **Глоссарий** (`<dl>`) для секций с терминами
- **Стильные таблицы** с моно-шрифтом в заголовках
- **Рисунки** встроены как base64 PNG (self-contained, без внешних файлов)
- **Примечания** — callout-блоки с цветной границей
- **ГОСТ/СП** — зелёные моноширинные теги; **[N]** — ссылки на библиографию
- **Формулы** — `<div class="eq">` с номером
- **Адаптивность**: на мобильных TOC сворачивается наверх

## Установка

```bash
# Требуется Python 3.10+
pip install PyMuPDF
```

## Использование

### Базовый запуск (все PDF из папки input/)

```bash
cd C:\HTMLart\pdf-to-html
python run.py
```

Положите PDF-файлы в папку `input/`, запустите `run.py`, HTML-файлы
появятся в `output/`. Промежуточные JSON-файлы (структурированные данные)
сохраняются в `work/`.

### Перегенерировать (игнорировать существующие)

```bash
python run.py --force
```

### Один файл

```bash
python run.py --file мой_документ.pdf
```

### Очистить промежуточные файлы

```bash
python run.py --clean
```

## Как это работает (подробно)

### Шаг 1: pdf_extract.py

Открывает PDF через PyMuPDF и извлекает:

- **Метаданные**: имя файла, количество страниц, заголовок (первая строка)
- **Текст**: `get_text("text")` по всем страницам, нормализация Unicode
  (математические italic-буквы → ASCII, Korean Hangul artifact → Latin),
  объединение строк в абзацы
- **Разделы**: поиск по regex `^(\d{1,2})\s+([А-ЯЁ]...)$` + named sections
  (Предисловие, Введение, Библиография). Фильтрация ложных срабатываний
  (ГОСТ/СП номера, clause-continuations, формулы)
- **Таблицы**: `page.find_tables()` — определение границ таблицы,
  извлечение ячеек, очистка от переносов строк
- **Рисунки**: поиск подписей «Рисунок N.M», определение области графика
  через `get_drawings()` + image blocks, рендер crop в PNG 3× zoom,
  кодирование в base64
- **Глоссарий**: для секций «Термины и определения» — парсинг
  «N.M term: definition» в список терминов

Результат: JSON файл в `work/`.

### Шаг 2: html_gen.py

Читает JSON и генерирует HTML:

- **TOC**: список разделов с якорными ссылками
- **Header**: eyebrow + title + auto-generated TL;DR
- **Разделы**: для каждого раздела — `<section>` с заголовком
  - **Термины** → `<dl class="glossary">` с терминами
  - **Библиография** → `<ol class="biblio">` с нумерацией [N]
  - **Нормативные ссылки** → `<ul class="ref-list">` с ГОСТ/СП
  - **Обычные разделы** → auto-detect подразделов (5.1, 5.2...) и
    под-подразделов (5.2.1, 5.2.2...) из clause-номеров, рендер в
    иерархические `<details>`
- **Таблицы**: `<figure class="tbl">` с заголовком и данными
- **Рисунки**: `<figure class="fig">` с base64 PNG

Результат: один .html файл в `output/`.

### Шаг 3: run.py

Оркестратор:

1. Сканирует `input/` на `.pdf` файлы
2. Для каждого: запускает `pdf_extract.py` → `html_gen.py`
3. Пропускает уже обработанные (если не `--force`)
4. Выводит статистику

## Ограничения

- **Универсальный парсер** — не идеален для всех типов документов.
  Нормативы с numbered sections (1, 2, 3...) парсятся хорошо. Документы
  без нумерации (статьи, презентации) — хуже (весь текст попадает в одну секцию).
- **Таблицы** — PyMuPDF `find_tables()` работает не со всеми PDF
  (особенно если таблицы отрисованы линиями, а не ячейками).
- **Рисунки** — если подпись «Рисунок N.M» не найдена, рисунок не извлекается.
- **Формулы** — извлекаются как текст (не LaTeX). Сложные формулы могут
  отображаться некорректно.
- **Шрифты** — если PDF использует нестандартные шрифты, Unicode-нормализация
  может не покрыть все символы.
- **Размер** — base64-картинки увеличивают размер HTML (~3.5 МБ на 9 рисунков).

## Настройка

### Изменить цветовую схему

В `html_gen.py` найдите блок `:root` в переменной `CSS` и поменяйте
oklch-значения `--accent`, `--bg`, `--ink` и т.д.

### Изменить шрифты

В `html_gen.py` найдите `--font-ui` и `--font-mono` в переменной `CSS`.

### Добавить named sections

В `pdf_extract.py` найдите список `NAMED_SECTIONS` и добавьте свои:
```python
NAMED_SECTIONS = [
    "Предисловие", "Введение", "Содержание", "Библиография",
    "Приложение А", "Приложение Б", "Приложение В",
    "Заключение", "Список литературы",  # ваши
]
```

### Обработать документы на английском

Замените в `pdf_extract.py` regex для section detection:
```python
section_re = re.compile(
    r"^(\d{1,2})\s+([A-Z][^\n]{2,80})$",
    re.MULTILINE,
)
```
И в `NAMED_SECTIONS` добавьте: `"Introduction"`, `"References"`, `"Appendix A"` и т.д.

- [Komplid](https://github.com/0odonkihooto0/Komplid2) — дизайн-токены (oklch, Inter, JetBrains Mono)
